## ğŸš€ Setup Development Environment

### Step 1: Clone Repository
```bash
git clone https://github.com/RenggoPandora/simantesa.git
cd simantesa
```

### Step 2: Install Backend Dependencies
```bash
# Install Composer packages
composer install
```

**Packages penting yang akan terinstall:**

- `laravel/fortify` - Authentication
- `inertiajs/inertia-laravel` - Inertia.js server adapter

### Step 3: Install Frontend Dependencies
```bash
# Install NPM packages
npm install
```

**Packages penting yang akan terinstall:**
- `react` & `react-dom` - UI library
- `@inertiajs/react` - Inertia.js client adapter
- `pdfjs-dist` - Client-side PDF processing
- `tailwindcss` - CSS framework

### Step 4: Environment Configuration
```bash
# Copy environment file
cp .env.example .env

# Generate application key
php artisan key:generate
```

### Step 5: Configure Database
Edit file `.env` dan sesuaikan dengan konfigurasi MySQL Anda:

```env
APP_NAME=SIMANTESA
APP_ENV=local
APP_DEBUG=true
APP_URL=http://127.0.0.1:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=simantesa
DB_USERNAME=root
DB_PASSWORD=your_password
```

**Buat database:**
```bash
# Login ke MySQL
mysql -u root -p

# Buat database
CREATE DATABASE simantesa CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit;
```

### Step 6: Run Database Migrations
```bash
# Fresh install dengan sample data (recommended untuk development)
php artisan migrate:fresh --seed

# Atau migration saja tanpa sample data
php artisan migrate
```

**Seeder akan membuat:**
- 1 Super Admin (admin@simantesa.com)
- 4 Perangkat Desa
- 27 Aset desa
- 14 Project dengan berbagai status
- 40+ Transaksi keuangan dengan bukti file

### Step 7: Setup Storage Symlink
```bash
# Create symbolic link untuk storage
php artisan storage:link
```

Ini akan membuat symlink dari `public/storage` ke `storage/app/public` agar file upload dapat diakses public.

### Step 8: Build Frontend Assets


**Catatan**: Saat pertama kali `npm run dev` atau `npm run build`, script otomatis akan menyalin `pdf.worker.min.mjs` dari `node_modules/pdfjs-dist/build/` ke folder `public/`. File ini diperlukan untuk fitur PDF processing.

### Step 9: Start Development Server
Buka terminal baru dan jalankan:

**Aplikasi akan berjalan di**: http://127.0.0.1:8000

2. Atau ubah Document Root melalui cPanel â†’ "Domains" â†’ "Document Root" â†’ arahkan ke `/home/username/simantesa/public`

**Jika menggunakan Nginx:**

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/simantesa/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

## ğŸ” Troubleshooting

### Build Issues
**Problem**: `npm run build` gagal atau PDF worker tidak tercopy

**Solution**:
```bash
# Manual copy PDF worker
cp node_modules/pdfjs-dist/build/pdf.worker.min.mjs public/pdf.worker.min.mjs

# Rebuild
npm run build
```

### PDF Upload Issues
**Problem**: Upload PDF gagal atau tidak terkonversi

**Solution**:
1. Check browser console untuk error PDF.js
2. Pastikan `public/pdf.worker.min.mjs` exists
3. Clear browser cache
4. Verify file size tidak melebihi upload limit

### Storage Link Issues
**Problem**: Bukti file tidak muncul (404 error)

**Solution**:
```bash
# Remove existing link
rm public/storage

# Recreate link
php artisan storage:link

# Verify
ls -la public/storage
```

### Permission Issues
**Problem**: 500 error atau "Permission denied"

**Solution**:
```bash
# Fix ownership
sudo chown -R www-data:www-data /var/www/simantesa

# Fix permissions
sudo chmod -R 755 /var/www/simantesa
sudo chmod -R 775 storage bootstrap/cache
```

### Recommended Hosting Providers
- **Shared Hosting**: Hostinger (Pro/Business plan), Niagahoster, Rumahweb
- **VPS**: DigitalOcean, Linode, Vultr, AWS Lightsail
- **Cloud**: AWS EC2, Google Cloud Compute Engine
- **Platform**: Laravel Forge, Ploi, Laravel Cloud

**Catatan**: SIMANTESA compatible dengan shared hosting karena PDF processing dilakukan di client-side, tidak memerlukan Ghostscript atau Imagick.
